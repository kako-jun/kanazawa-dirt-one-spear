## 3. キャラクター設計

### 3.1 コンセプト

**AI・統計解説をキャラクターで親しみやすく**

難しいアルゴリズムや統計の説明を、キャラクターの会話形式で分かりやすく伝える。

---

### 3.2 キャラクター設定

#### メインキャラクター: アヤメ（分析担当・成年女性）

**名前**: アヤメ（統計の"綾目"から）

**ビジュアル**:
- かわいい成年女性（20代前半）
- メガネっ娘（丸メガネ）
- 研究者風の白衣 or カジュアルな服装
- ノートPCやグラフを持っている
- 表情: 真面目だけど親しみやすい

**性格**:
- 真面目で分析好き
- データが大好き
- 難しいことも分かりやすく説明してくれる
- ちょっと天然なところも

**セリフ例**:
```
「統計学って難しそう？でも大丈夫！
 図を使って順番に説明していくね」

「p値が0.05以下だから、これは偶然じゃないって
 言えるんだよ。つまりね...」

「LightGBMっていうのはね、決定木っていう
 木みたいな構造をたくさん作って...」
```

---

#### AI擬人化キャラクター: アルゴ（性別不明）

**名前**: アルゴ（アルゴリズムから）

**ビジュアル**:
- 性別不明・中性的
- 近未来的な服装（フード付き？）
- デジタル的な装飾（回路基板風）
- 髪色: 銀色やブルーなど非現実的な色
- 表情: クールだけど優しい

**性格**:
- 冷静沈着
- 論理的
- でも時々人間らしい疑問も持つ
- アヤメの相棒

**セリフ例**:
```
「僕が予想したのは3-7-5だ。
 理由？それは...」

「統計的に有意な差が見られた。
 つまり、これは意味のある結果だよ」

「ディープラーニングの僕は
 なぜそう判断したか説明できない。
 でも、それが強みでもあるんだ」
```

### 3.4 配置場所

**解説ページ**:
- `/learn/statistics` - 統計学の基礎
- `/learn/ai` - AIの仕組み
- `/learn/validation` - 検証方法
- `/learn/features` - 特徴量とは

**予想ページ内**:
- 「この予想の根拠を見る」→ アヤメが解説
- 「AIの仕組み」→ アルゴが説明

**統計ページ内**:
- グラフの見方 → アヤメが補足説明

## 4. 馬券購入促進機能

### 4.1 プロジェクトの使命

### 4.4 法的・倫理的配慮

#### 注意事項の明記
- 「ギャンブル依存に注意」
- 「20歳未満は購入できません」
- 「余剰資金で楽しみましょう」
- 「必ず当たるわけではありません」

#### 適正な表現
- ❌ 「絶対当たる」「必ず儲かる」「この予想を買って」
- ⭕ 「的中の可能性」「期待値が高い」「応援しよう」「参考情報としてご覧ください」

#### ⚠️ 重要な制約

**AI予想の3連単について**:
- 「この予想を買ってください」とは言わない
- 「参考情報」「データ分析結果」として提示する
- ユーザー自身の判断を尊重する
- 「楽天競馬で馬券を買う」ボタンは、予想とは別の文脈で表示（「金沢競馬を応援する」という一般的な訴求）

**正しい表現例**:
- ⭕ 「AI予想は参考情報です。自己責任でご判断ください」
- ⭕ 「データから導き出した一つの見解です」
- ⭕ 「金沢競馬を楽しむ際の参考にしてください」
- ⭕ 「馬券購入は楽天競馬で。100円から金沢競馬を応援できます」

**避けるべき表現**:
- ❌ 「この3連単を買おう！」
- ❌ 「予想通りに買えば当たる」
- ❌ 「今すぐこの組み合わせを購入」

**実装内容**:
- D3.jsやReact Flowを使った血統ツリーの可視化
- 父系・母系を色分け表示
- クリックで祖先の詳細情報を表示
- インブリード（近親交配）の検出・表示

**技術的検討事項**:
- 現在のデータ構造: 血統の馬名（文字列）のみ抽出済み
- 完全な家系図には馬名→馬IDの紐付けが必要
- **重要な課題**: 金沢競馬で走った馬の親は、中央競馬や他の地方競馬出身が多い
  - → `horses`テーブルには存在しないケースが大半
  - → 血統馬マスターテーブル（`pedigree_horses`）の作成が必要
  - → または文字列ベースで表示（ID参照なし）

**実装案**:
1. **簡易版**: 文字列ベースで4代血統を表示（DB参照なし）
   - 実装容易
   - リンクなし、統計なし
2. **完全版**: 血統馬マスターを作成し、自己参照で無限に辿れる
   - 実装コスト高
   - 中央競馬の馬データ取得が必要
   - 再帰的クエリで祖先を無限に辿れる

**優先度**: 中〜低（Phase 3で検討）

**メモ**: 2025-11-15 - ユーザーからの提案により追加
