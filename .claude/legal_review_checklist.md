## NAR公式サイト利用規約の要点

### 確認日: 2025-11-11

**利用規約URL:** https://www.keiba.go.jp/terms.html

**重要条項:**

1. **著作権**
   > 「当サイトで使用している画像及びテキスト等の掲載情報についての全ての権利は、原則として地方競馬全国協会に帰属します。**事前の許諾なく転載、複製することを禁じます。**」

2. **robots.txt**
   - 存在しない（404）
   - 明示的なクローリング制限なし

3. **スクレイピング禁止条項**
   - 明示的な記載なし
   - ただし著作権条項により間接的に制限

4. **免責事項**
   - 情報の正確性・有用性を保証しない
   - 予告なしの変更・削除あり

## 実装時チェックリスト

### 開発初期（設計段階）

- [ ] サイトのコンセプトを決定（予想特化 or 統計分析 or 結果DB？）
- [ ] 「公式サイトの代替」にならないか検討
- [ ] 主要機能がグレーゾーンに該当しないか確認
- [ ] 利用規約ページの文案作成（非営利・研究目的明記）

### 実装中（機能追加時）

**新機能追加の都度確認:**

1. **表示内容チェック**
   - [ ] NAR公式と同じ形式・構成になっていないか
   - [ ] 全レース結果を一覧表示していないか
   - [ ] 検索で全件閲覧可能になっていないか

2. **出典表記チェック**
   - [ ] ©地方競馬全国協会（NAR）の表記あるか
   - [ ] 「データ出典: NAR公式サイト」の記載あるか
   - [ ] 公式サイトへのリンクあるか

3. **引用の要件チェック（引用する場合）**
   - [ ] 自分の分析が「主」、公式データが「従」の関係か
   - [ ] 引用部分が明確に区別されているか（「」囲み、色分けなど）
   - [ ] 引用の必要性があるか（統計値だけでは不十分か）
   - [ ] 全体の分量に対して引用が過剰でないか

4. **営利性チェック**
   - [ ] 広告を掲載していないか
   - [ ] 有料会員制になっていないか
   - [ ] データ販売をしていないか
   - [ ] 「非営利・研究目的」の明記があるか

### デプロイ前（公開前）

- [ ] 全ページで著作権表記を確認
- [ ] 利用規約ページの設置
- [ ] プライバシーポリシーの設置
- [ ] お問い合わせフォームの設置（削除要請対応用）
- [ ] NAR公式へのリンク一覧

### 公開後（運用中）

- [ ] NAR公式サイトの利用規約更新を定期確認（年1回）
- [ ] 削除要請があった場合の対応フロー整備
- [ ] アクセス過多によるNAR公式への負荷確認
- [ ] ユーザーからの通報対応

## 具体的な実装例（OK/NG）

### ❌ NG例1: レース結果の完全転載

```html
<h2>2025年4月6日 第1レース 結果</h2>
<table>
  <tr><th>着順</th><th>馬番</th><th>馬名</th><th>タイム</th></tr>
  <tr><td>1</td><td>7</td><td>フェスティヴノンノ</td><td>1:30.9</td></tr>
  <tr><td>2</td><td>9</td><td>ベルウッドボヌール</td><td>1:31.2</td></tr>
  ...
</table>
<p>三連単配当: 7-9-1 74,490円</p>
```

**問題点:**
- NAR公式と同じ情報を同じ形式で表示
- 完全転載に該当
- 出典表記があっても「引用」の範囲を超える

---

### ⚠️ グレー例: 独自カラム追加

```html
<h2>2025年4月6日 第1レース 結果（AI予想的中！）</h2>
<table>
  <tr><th>着順</th><th>馬番</th><th>馬名</th><th>AI予想順位</th><th>的中</th></tr>
  <tr><td>1</td><td>7</td><td>フェスティヴノンノ</td><td>1位</td><td>✅</td></tr>
  ...
</table>
<small>出典: NAR公式サイト</small>
```

**リスク:**
- 独自カラムがあるが、基本は公式データそのまま
- 「加工」と言えるか微妙
- 全レースでこれをやると実質DB公開

**改善案:**
- 統計値のみ表示（個別馬名は非表示）
- または1レースのみ例示（引用）

---

### ✅ OK例1: 統計サマリー

```html
<h2>2025年4月6日 第1レース 分析</h2>
<p>AI予想: 7-9-1（信頼度85%） → <strong>的中！</strong></p>
<p>配当: 74,490円（平均の3.2倍）</p>
<p>1番人気: 2番（外れ）</p>
<p>馬場: 重（不利条件）</p>

<h3>統計データ</h3>
<ul>
  <li>このレースでの平均配当: 23,145円</li>
  <li>重馬場での1番人気勝率: 28.5%</li>
  <li>AI予想の的中率（過去30日）: 62.3%</li>
</ul>

<a href="https://www.keiba.go.jp/...">詳細はNAR公式サイトへ</a>
<small>データ出典: ©地方競馬全国協会</small>
```

**良い点:**
- 統計値・分析が中心
- 個別レース結果は最小限
- 公式へ誘導
- 出典明記

---

### ✅ OK例2: 引用として使用

```html
<h2>金沢競馬の馬場状態と配当の関係</h2>

<p>2015-2025年のデータを分析した結果、馬場が「重」「不良」の場合、
平均配当が1.8倍に増加することが分かりました。</p>

<h3>分析例</h3>
<blockquote>
  2025年4月6日 第1レース（馬場: 重）
  三連単配当: 7-9-1 74,490円
  <cite>出典: NAR公式サイト</cite>
</blockquote>

<p>この日の平均配当は23,145円なので、3.2倍の高配当でした。
このように、馬場悪化時は荒れやすい傾向があります。</p>
```

**良い点:**
- 分析が主、公式データは例示（従）
- blockquoteで引用を明確化
- 出典表記あり
- 必要最小限の引用

## まとめ

**基本方針:**
1. **統計・分析特化**（生データ転載NG）
2. **公式への誘導**（代替サービス化NG）
3. **非営利運営**（営利目的NG）
4. **透明性**（出典明記、問い合わせ対応）
5. **謙虚さ**（NAR公式への敬意）

**判断に迷ったら:**
→ このチェックリストで「🟢 安全」に該当するか確認
