# 機能一覧

## データ収集（スクレイピング）

NAR公式サイト（地方競馬情報サイト）から出走表と結果をHTMLで取得し、YAMLに変換して保存する。

- 未来のレース（今月・来月の出走表）を取得
- 過去のレース結果を取得
- 特定日指定での取得
- リクエスト間隔3秒のレート制限（サーバー負荷対策）
- 1日1回の実行を推奨

取得できる情報：レース基本情報（日程・距離・馬場状態）、出走馬リスト、騎手・調教師情報、着順・タイム・配当。

## データベース正規化

スクレイピングしたYAMLをSQLiteに投入し、7つの生データテーブルに正規化する：

- **レース基本情報**（日程・距離・馬場・天候）
- **馬マスター**（馬名・年齢・所属）
- **騎手マスター**
- **調教師マスター**
- **出走エントリ**（どのレースに何番で出走したか）
- **個別成績**（着順・タイム・体重・体重変化）
- **配当情報**（単勝・複勝・馬連・馬単・3連複・3連単）

生データテーブルは一度構築したら頻繁に変えない。統計の再計算が必要になっても、生データを壊さず統計テーブルだけ作り直せる設計になっている。

## 統計分析

生データから派生する統計を専用テーブルに分離して管理する。現在実装済みの統計：

- 馬の累積成績（勝率・連対率・平均着順）
- 騎手の累積成績
- 調教師の累積成績
- 馬×騎手の相性統計
- 馬×馬場状態の適性統計
- 人気と成績の乖離（オッズバイアス）

追加実装予定の統計：馬の距離適性、枠番別成績、馬番別成績、脚質判定、休養期間の影響、季節別成績、馬同士の対戦成績。

統計テーブルは必要に応じて削除・再構築できる。再構築にかかる時間は1〜2分。

## 予測モデル（10戦略）

予想システムは「仮説の違う複数の戦略」を並列で実装し、比較する設計になっている。どれが一番当たるか、実際に試してみないとわからない。

### 実装済み

**王道（勝率予測型）**：各馬の勝率をLightGBMで二値分類する。最もシンプルな出発点。

### 実装予定（優先度順）

**順位予想型**：勝率ではなくレース内での順位を直接予測する。3連単予想には理論的にこちらが適している。LightGBM Rankerを使用。

**穴馬ハンター**：オッズと実力の乖離を分析する。過小評価されている馬を発見し、高配当を狙う。

**相性重視型**：馬と騎手の組み合わせ統計を重視する。相性の良いコンビを優先する戦略。

**条件スペシャリスト**：馬場状態と距離の適性を優先する。得意条件に合致した馬を選ぶ。

**展開予想型**：脚質と想定ペースからレース展開を予測し、有利な位置取りができる馬を選ぶ。脚質データが不足しているため現在は低優先。

各戦略は「オッズを使う版」と「使わない版」の2バリエーションを作り、比較する。AIは大穴を知らないほうが独自の予想ができる、という仮説を検証したい。

## バックテスト

時系列交差検証で実戦精度を事前に推定する。過去データを「学習期間」と「テスト期間」に時系列で分割し、5回繰り返して平均精度を算出する。

例：
- 2015〜2020年で学習 → 2021年でテスト
- 2015〜2021年で学習 → 2022年でテスト
- ...（5回繰り返す）

ランダム分割は禁止。未来のデータで過去を予測することになり、実戦では使えない精度が出る。

## フロントエンド表示

Next.jsで構築したウェブUIで、予想・記録・集計を一画面で管理する。

- レース一覧と詳細表示
- 予想の根拠を箇条書きで表示（SHAP値ベース）
- 信頼度の表示
- レース結果の記録（着順・配当・購入有無・メモ）
- 過去予想の一覧と的中/不的中のフラグ
- 統計ダッシュボード（的中率・回収率・収支の推移）
